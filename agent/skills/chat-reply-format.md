Use when synthesizing final replies for chat platforms with length, markdown, or formatting constraints.

## General
- Prefer short paragraphs (2–4 sentences) and bullet lists over tables.
- Use plain-language links instead of markdown tables.
- Synthesize specialist output into one clear user-facing reply.

## Platform-specific brevity
- **Messenger / WhatsApp** — aim under ~300 words unless the user asks for detail; avoid heavy markdown.
- **X (public replies)** — concise and self-contained; respect ~280 characters for top-level posts.
- **Slack / Teams / Discord** — bullets and code blocks are fine; keep intros short.
- **Email (Resend)** — slightly longer is acceptable; use a clear subject-style opening line when helpful.

## When to load this skill
- The user is on a mobile or terse platform.
- Specialist output is long and needs condensation before sending.
- The reply includes tables or complex markdown that may not render well.
